# Contributing Guidelines

First off, thank you for considering contributing to this project! It's people like you that make this project better for everyone. 

To ensure a smooth collaboration process, please follow the steps below to fork the repository, make your changes, and submit a Pull Request (PR).

---

## 1. Fork the Repository
Click the **Fork** button at the top right corner of this repository's page. This will create a copy of the project in your own GitHub account where you can safely make changes.

## 2. Clone Your Fork
Clone the forked repository to your local machine to start working on it. Open your terminal and run the following commands:

```bash
# Replace <your-username> and <repository-name> with your actual details
git clone [https://github.com/](https://github.com/)<your-username>/<repository-name>.git
cd <repository-name>
```
## 3. Create a New Branch
Always create a new branch for your changes rather than working directly on the *main* or *master* branch. Use a descriptive name for your branch that reflects the feature or fix you are working on:

```bash
git checkout -b feature/your-feature-name
```

## 4. File Naming Rules
Before you add any new files, please ensure they adhere to our project's strict naming conventions. Pull requests containing incorrectly named files will not be reviewed or merged.

* Example (Good): `wireless_networking_2020_mid_Term.pdf`
* you can use pdf/Image/JPG/JPEG

## 5. Make Changes and Commit
Make your modifications, add your new files, and commit them with clear, concise commit messages. A good commit message helps reviewers understand what you added.

```bash
git add .
git commit -m "Add short, descriptive message of the feature or fix"
```

## 6. Push to your work
Push your new branch and changes up to your forked repository on GitHub:
```bash
git push origin feature/your-feature-name
```

## 7. Create a Pull Request
1. Navigate back to the original repository on GitHub.

2. You should see a prompt at the top of the page indicating your recently pushed branch. Click the Compare & pull request button.

3. If you don't see the prompt, go to the Pull requests tab and click New pull request.

4. Ensure the base branch is the original repository's main branch, and the compare branch is the feature branch from your fork.

5. Add a clear title and description to your PR explaining what changes you made, how you tested them, and any related issue numbers.

6. Click Create pull request.

7. Once submitted, project maintainers will review your changes. Be prepared to make further tweaks if requested. 

## Thank you for your contribution!
